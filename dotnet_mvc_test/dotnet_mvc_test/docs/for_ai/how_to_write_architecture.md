# `.ai\knowledge\general\architecture.md`に記載する内容の書き方

常に全体の流れ・関連が分かりやすい書き方にしてください。

- ## Good
    - 抽象化されたASCII図を用いて、全体の構成や流れを説明している
    - 各コンポーネントの役割や責務が明確に説明されている
    - それぞれの設計が用いられた理由や背景が簡潔に用語を使って（例：マイクロサービスアーキテクチャ、CQRS、イベントソーシングなど）説明されている
    - 将来的な拡張性や変更に対する考慮点が記載されている

- ## Bad
    - コードレベルの詳細に過度に踏み込んでいる、コード例が含まれている
    - あまりにも長いドキュメント(最大400行を超えるなど)
    - Tree構造がそのまま記載されている
        - Tree構造そのものでなく、要点をまとめたASCII図などであればOK
    - あまりにも現状の実装に依存した内容
        - 理想的なアーキテクチャや設計思想を中心に記載した上で、乖離点や現状の問題点を補足的に記載する形にする
